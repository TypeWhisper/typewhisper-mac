# TypeWhisper 1.0 Readiness

TypeWhisper `1.0` ist ein stabiler Direct-Download-Release fuer macOS. Der Mac App Store ist fuer diese Release-Definition komplett out of scope. Vor `1.0` werden keine grossen neuen Produktflaechen mehr hinzugefuegt; Fokus ist Verlaesslichkeit, klare Produktgrenzen, Upgrade-Sicherheit und schnelle Support-Diagnose.

## Zielgruppe

- macOS-Nutzer, die systemweite Dictation mit eigener Engine-Wahl wollen
- Nutzer, die Audio- oder Videodateien lokal oder per API transkribieren wollen
- Power-User, die Prompt-Verarbeitung, Profiles und lokale Automatisierung brauchen

## Offiziell unterstuetzter Kern

- Systemweite Dictation mit globalem Hotkey und Texteinfuegung
- Datei-Transkription inklusive Batch-Verarbeitung und Export
- Prompt-Verarbeitung mit mitgelieferten Presets und eigenen Aktionen
- Profiles fuer App- und URL-basierte Steuerung
- History, Dictionary und Snippets
- Gebuendelte Standard-Integrationen und Standard-Plugins

## Advanced Surfaces in 1.0

Diese Flaechen bleiben Bestandteil von `1.0`, werden aber als Advanced-/Automation-Surfaces positioniert:

- Lokale HTTP API unter `/v1/*`
- `typewhisper` CLI
- Plugin SDK und Plugin-Manifeste
- Widgets
- Watch Folder

## Nicht-Ziele vor 1.0

- Kein Mac-App-Store-Launch und keine App-Store-spezifischen Gates
- Keine neuen grossen Engines, Plugin-Klassen oder oeffentlichen API-Familien
- Keine Breaking Changes an dokumentierter HTTP API, CLI oder Plugin SDK
- Keine breit angelegte UI-Automation als Voraussetzung fuer `1.0`

## Stabilitaetsvertraege fuer 1.x

### HTTP API

- Dokumentierte Endpunkte unter `/v1/*` bleiben fuer `1.x` stabil.
- Response-Felder duerfen vor `2.0` nicht ohne Deprecation entfernt werden.
- Die API ist loopback-only, standardmaessig deaktiviert und fuer lokale Automatisierung gedacht.

### CLI

- `typewhisper status`
- `typewhisper models`
- `typewhisper transcribe`
- Flags: `--port`, `--json`, `--language`, `--task`, `--translate-to`

### Plugin SDK

- `manifest.json`
- `TypeWhisperPlugin`
- `PostProcessorPlugin`
- `LLMProviderPlugin`
- `TranscriptionEnginePlugin`
- `ActionPlugin`
- `HostServices`

## Release Gates

`1.0.0` wird nur getaggt, wenn alle folgenden Bedingungen erfuellt sind:

- `xcodebuild test` fuer den App-Scheme ist gruen.
- `swift test --package-path TypeWhisperPluginSDK` ist gruen.
- Release-Build der App ist gruen.
- Es gibt keine First-Party-Build-Warnungen.
- Plugin-Manifeste validieren erfolgreich.
- README, Security-Hinweise und Support-Matrix sind aktuell.
- Ein `1.0.0-rc1` lief mehrere Tage auf echten Maschinen ohne P0/P1-Blocker.
- `1.0.0-rc*` wird als GitHub-Prerelease fuer manuellen Download verteilt, aktualisiert weder Sparkle-Appcast noch Homebrew und bleibt als installierter RC-Build aus dem Stable-Auto-Update-Kanal heraus.

## Manuelle Smoke-Checks vor dem Tag

- Fresh install auf einer sauberen Maschine
- Rechtefluss: Mikrofon, Accessibility, Recovery nach entzogenem Zugriff
- Erste erfolgreiche Dictation
- Datei-Transkription
- Prompt-Aktion aus der App
- History edit/export
- Profile-Matching fuer App und URL
- Plugin enable/disable
- CLI gegen laufenden lokalen Server
- HTTP API `status`, `models`, `transcribe`
- Upgrade von `0.14.x` mit Erhalt von History, Profiles, Dictionary, Snippets, Hotkeys und Plugin-Status

## Support und Diagnostik

- Support-Anfragen sollen immer den JSON-Diagnose-Export aus dem Error Log enthalten.
- Der Export enthaelt Version, macOS-Version, API-Status, aktive Plugins und privacy-sichere Settings-Metadaten.
- Der Export enthaelt keine API-Keys, keine Audiodaten und keine Transkript-Historie.
